## 2. Discoveries

- **The librarian could certify a STALE shelf.** `~/.claude/skills` is a git work tree and skills get authored on OTHER machines. If local is behind `origin/main`, the audit's count is a lie and the index gets written short. This is not hypothetical — `page-rethink` (authored by Brent 2026-07-11 on another box, commits `e1ece53`/`4352a5a`/`186d788`) was absent locally; the audit called the shelf clean and the index shipped one skill short. **This was the session's real bug.**
- **Claude Code namespaces credentials PER CONFIG DIR.** macOS Keychain holds `Claude Code-credentials-<hash>` entries, one per `CLAUDE_CONFIG_DIR`. So **multiple accounts stay logged in simultaneously** — no logout/login dance. Verified: a fresh config dir reports "Not logged in" rather than inheriting the default's auth.
- **`codex-skills/skills/` holds 58 duplicate `SKILL.md` copies** — adapted copies of the shelf, living inside the shelf's own backup repo. Claude Code does NOT load them (no top-level `SKILL.md`, scan is one level deep), so no runtime harm — but it's a drift trap and the reason for the split decision.
- **`codex-skills/scripts/build.py` already accepts `--repo`** (it merely DEFAULTS to `parents[2]` = the skills dir). So the split costs a flag, not a rewrite.
- **Memory is scoped by WORKING DIRECTORY, not by repo.** Brent assumed per-repo. Four scopes exist; `-Users-brentbryson` (home) is the global one loaded in nearly every session.
- **GitHub `warm-import` repo is ARCHIVED (read-only)** — `webapp/reclassify.py` + a session doc are uncommitted there and can never be pushed without unarchiving.
- The "11 orphan sync branches" I nearly reported **did not exist on GitHub** — `backup.sh` had already deleted each at merge. They were stale local remote-tracking refs; `git remote prune` cleared them.
